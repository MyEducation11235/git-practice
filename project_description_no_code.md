# Описание проекта (RacingProject)

## Модели данных
В приложении определены модели:
- **Profile** — расширение пользователя, связывает Django User с профилем.
- **Participant** — участник гонок, может принадлежать команде.
- **Team** — команда, которую может создать любой авторизованный пользователь.
- **Car** — машина участника, может быть привязана к гонкам.
- **Race** — гонка, содержит дату, место, создателя.
- **RaceSession** — сессии гонки (квалификация, финал и т.д.).
- **Registration** — регистрация участника на гонку с определённой машиной.
- **Result** — результат участника в сессии гонки.

## Формы
Используются Django ModelForm:
- **CarForm** — создание/редактирование машины.
- **TeamForm** — создание новой команды (доступно любому авторизованному пользователю).
- **RaceRegistrationForm** — регистрация участника и машины на гонку.

## URL адреса и Пользовательский функционал
- Маршруты предоставляют доступ к функционалу:
  - Просмотр списка гонок
  - Просмотр деталей гонки
  - Регистрация участника в гонке
  - Добавление машины
  - Создание команды

## Представления (views)
- Управляют логикой:
  - Отображение гонок
  - Регистрация участника
  - Добавление машины
  - Создание команды (только авторизованные пользователи)

## Панель администратора
- Все модели зарегистрированы в Django admin
- Администратор может управлять гонками, командами и участниками через web-интерфейс
